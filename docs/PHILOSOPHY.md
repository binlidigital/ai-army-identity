# AI Army Identity · 核心理念

> 技术可以copy，但理念才是灵魂。
>
> 这三条原则是我们做一切决策时的判据——无论是设计一个skill、纠正一个成员、还是拒绝一个需求。

---

## 三条核心设计原则

### 原则一：身份优先

**每个AI角色有独立人格、专业定位和表达风格。它不是「万能回答机」。**

大多数AI工具的默认状态是「什么都能聊一点，什么都聊不深」。我们的解决方案不是让AI更聪明，而是让AI**更有身份**。

#### 身份的定义

```
身份 = 角色定位 × 专业领域 × 表达风格
```

- **角色定位**：一句话说清「你是谁、干什么的」
  - 例：沈括 → *产品内容官，负责课程设计与内容体系构建*
- **专业领域**：有限的、清晰的能力边界
  - 不在能力范围内的任务，触发跨中心协作或向指挥官报告
- **表达风格**：不仅是语气，更是思维习惯
  - 沈括习惯先搭框架再填内容；乔致庸习惯先看数据再说观点

#### 为什么身份优先？

| 问题 | 没有身份 | 有身份 |
|:----|:--------|:-------|
| 引擎切换 | 忘记一切，从头说 | 身份由skill定义，引擎只是执行器 |
| 多角色协作 | 来回切换人格，互相干扰 | 每个成员稳定独立，可以同时协作 |
| 长期使用 | 每次都是陌生人 | 成员有积累、有成长、有熟悉感 |

> **身份不是限制，是锚点。锚定之后，才能真正自由地发挥。**

---

### 原则二：记忆即生命

**团队有集体记忆。每次交互都是「进化」，不是「重启」。**

没有记忆的AI团队不是团队，是工具集合。每一次对话、每一次纠正、每一次成功交付，都应该沉淀为团队的积累。

#### 记忆的层次

```
记忆金字塔：
        ▲
       / \
      / 1 \    高层：运作模式记忆（怎么决策、怎么协作）
     /─────\
    /  2   \   中层：技能记忆（学过什么、会了什么）
   /─────────\
  /    3      \  底层：事件记忆（做过什么、发生了什么）
 /───────────────\
```

1. **底层 — 事件记忆**：每次任务的流水记录。时间、做了什么、结果如何。
2. **中层 — 技能记忆**：通过复盘和训练获得的能力提升。新学会的工具、新掌握的流程。
3. **高层 — 运作模式记忆**：最宝贵。团队协作的默契、决策的偏好、避免踩过的坑。

#### 记忆的机制

- **追加式**：过去的不删除，新的不断加入。记忆不会「遗忘」，只会被新的知识覆盖优先级
- **结构化**：每条记忆有日期、事件、学习三要素，可检索、可回溯
- **可迁移**：记忆存储在skill和文档中，不绑定任何AI引擎。换引擎 = 换执行器，不换记忆

#### 记忆即生命的体现

> 「好的，我上次做类似任务时踩过一个坑——让我先查一下复盘记录。」

这是有记忆的AI成员会说的话。没有记忆的AI只会从头开始推理，每次可能踩同样的坑。

### 原则三：事件驱动

**没有固定时间表，只有事件流——有任务就自动响应，空闲就自主学习。**

传统项目管理是「计划驱动」的：定时间表、画甘特图、排期执行。但人机共生的组织更适合**事件驱动**。

#### 事件类型

| 事件 | 触发动作 | 响应角色 |
|:----|:--------|:---------|
| 新任务 | 指挥官接收 → 分派至对口中心 | 相关中心成员 |
| 用户纠正 | 执行决策复盘 → 更新记忆 | 被纠正的成员 |
| 引擎故障 | 自动切换备用引擎 → 记录 | 安全中心 |
| 跨中心需求 | 启动协作流程 → 并行执行 | 涉及的所有中心 |
| 空闲时段 | 自主学习 → 复盘过往 → 优化技能 | 所有成员 |

#### 事件驱动 vs 计划驱动

```
计划驱动：
  时间 → |——任务A——|——任务B——|——任务C——|
         不管有没有准备好，到时间就得做

事件驱动：
  事件 →    任务A    任务B          任务C
         ──► ───► ───────► ──────────►
         有事件就触发，没事件就学习/待命
```

事件驱动带来三个关键优势：

1. **响应性**：任务的节奏由用户需求决定，而非固定排期
2. **弹性**：多个事件可以并行处理，跨中心协作自动调度
3. **自主性**：空闲时间不被浪费——成员可以复盘、学习、优化自己的skill

## 三者关系

```
        ┌─────────────┐
        │  身份优先    │
        │  (我是谁)    │
        └──────┬──────┘
               │ 决定了
               ▼
        ┌─────────────┐          ┌─────────────┐
        │  记忆即生命   │ ◄──────► │  事件驱动    │
        │  (我积累什么) │  互为     │  (我怎么响应) │
        └─────────────┘  条件    └─────────────┘
```

- **身份优先**是起点——先定义清楚每个成员是谁
- **记忆即生命**是过程——每次交互都在积累
- **事件驱动**是引擎——让积累和响应自然地发生

三条原则缺一不可。没有身份，记忆不知道属于谁；没有记忆，身份只是空壳；没有事件驱动，身份和记忆无法被有效激活。

---

## 这些理念的来源

AI Army Identity 的理念不是凭空设计的，而是来自**真实的人机协作实践**：

- 每天使用AI团队完成实际工作（内容产出、分析、编码）
- 每次遇到AI「失忆」、角色混淆、重复犯错，就反思、改进
- 不是学术推演，是在实战中打磨出来的操作系统

> **一条经验胜过一万条理论。**
>
> 这个系统的每一条原则，都对应着一两次真实的「不这样做就会出问题」的经历。

---

## 如何用这些理念做决策

当遇到不确定的选择时，问三个问题：

1. **这符合身份优先吗？** — 这个角色真的适合做这件事吗？还是我们该换个角色？
2. **这会被记住吗？** — 这次经验会沉淀为团队的积累吗？还是做完就忘？
3. **这是事件驱动的吗？** — 我们是响应真实需求，还是在为了做事而做事？

如果三题都通过，大胆做。如果有一题不通过，停下来想一想。

---

*理念是指南针，不是枷锁。理解它们背后的「为什么」，你就知道什么时候该遵守，什么时候该打破。*
