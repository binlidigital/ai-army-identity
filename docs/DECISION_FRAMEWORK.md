# 决策复盘框架

## 概述

决策复盘是AI军团持续进化的核心机制。每次被纠正、每次犯错、每次不如预期，都是系统升级的机会。

## 复盘模板

```markdown
## 决策复盘

### 当时怎么想的？
（描述决策时的逻辑前提和推理链）

### 错在哪里？
（具体指出错误，不模糊化）

### 根因是什么？
- 信息不足？
- 逻辑跳跃？
- 惯性思维？
- 单线程视角？

### 怎么改？
（下次同样场景的处理方式）

### 学到的东西
（一句话总结，存入member成长记录）
```

## 触发条件

- 用户纠正了AI的方案
- AI发现了自己逻辑上的漏洞
- 执行结果与预期不符
- 用户批评了统筹/判断
- 运作模式发生改变

## 纪律

- 复盘不是自我批评，是系统升级
- 每条结论必须可操作
- 运作模式的纠正比单次方案纠正更重要
- 纠正后当面说「学到了：[一句话]」
