## 架构总览

AI Army Identity 采用**三层分离 + 横向技能扩展**的架构设计，确保引擎无关性、身份持久性和事件驱动能力。

## 三层架构详解

### 1. 应用层 / 交互层

用户直接面对的组织界面。以指挥官（Commander）为中心，下辖六大中心，形成矩阵式协作结构。

**核心职责：**
- **指挥官（萧总）**：接收用户意图 → 拆解任务 → 调度对口中心 → 质检 → 交付
- **六大中心**：产品、市场、技术、风控、安全、日志——各自维护专业上下文
- **协作流程**：跨中心任务以 `cross-center-collab.skill.md` 为标准流程并行执行

### 2. 身份映射层（核心）

整个系统的灵魂所在。**这是唯一不可替换的层**——它承载了所有AI成员的身份、记忆和学习能力。

**子模块：**

| 模块 | 载体 | 职责 |
|:----|:-----|:-----|
| 身份映射 | `ai-army-identity.skill.md` | 定义每个角色的 name / role / personality / expertise / memory / skills |
| 决策复盘 | `decision-review.skill.md` | 每次纠正后执行五步复盘——记录、归因、改进行动、存入记忆 |
| 跨中心协作 | `cross-center-collab.skill.md` | 多角色并行工作的调度与同步标准流程 |
| 引擎健康 | `engine-health.skill.md` | 自检引擎状态，故障时自动切换备用引擎 |

**数据流：**

```
事件触发 → 指挥官调度 → 身份映射加载 →
  ├─ AI成员A（含个性+记忆）→ 执行任务
  ├─ AI成员B（含个性+记忆）→ 执行任务
  └─ 结果汇总 → 质量检查 → 复盘（如有纠正）→ 记忆更新 → 归档
```

### 3. AI 引擎层（可替换）

底层AI能力提供者。通过身份映射层注入的 prompt 前缀，确保任何引擎以正确的「人格」执行任务。

**设计约束：**
- 引擎切换不影响AI成员的身份认知——身份由skill定义，不由引擎记忆
- 引擎故障触发 `engine-health.skill.md` 自动切换，用户无感知
- 当前支持：DeepSeek / OpenAI / Claude / Gemini

## 技能系统架构

Skill 是本系统的**一等公民**。每一个skill是一个标准化的Markdown文件，以 YAML front-matter 作为元数据头。

```
skill文件格式：
┌────────────────────────────┐
│ ---                        │
│ name: skill-name            │
│ description: ...            │
│ version: 1.0.0              │
│ author: ...                 │
│ license: Apache-2.0         │
│ ---                         │
│                             │
│ # 技能正文（Markdown）       │
│                             │
│ ## 触发条件                  │
│                             │
│ ## 执行模板                  │
│                             │
│ ## 输出规范                  │
└────────────────────────────┘
```

详见 [SKILL_SYSTEM.md](SKILL_SYSTEM.md)。

## 记忆系统架构

记忆是AI成员「活着」的证据。每个成员维护自己的成长记录表：

```
成员记忆结构：
├── 角色定义（不可变，由identity.skill定义）
├── 技能清单（可扩展，通过学习新增）
├── 成长记录表（追加式，每次复盘+1行）
│   ├── 日期
│   ├── 事件摘要
│   └── 学到的东西（一句话原则）
└── 协作记录（跨中心任务的参与历史）
```

## 关键设计决策

| 决策 | 选择 | 理由 |
|:----|:-----|:------|
| 引擎无关性 | Skill定义身份 | 换引擎不换人，记忆由框架维护而非引擎上下文 |
| 持久化方式 | Markdown文件 | 人类可读、版本可控、无需数据库 |
| 驱动范式 | 事件驱动 | 非定时任务，有事件才触发，空闲时自主学习 |
| 学习机制 | 决策复盘框架 | 每次纠正是一次系统升级，而非单次修复 |
| 扩展方式 | Skill文件热加载 | 新增能力 = 新增一个Markdown文件 |

## 部署视图

无需服务器、无需数据库、无需容器。项目本质是一套**人类可读的Markdown配置 + AI代理平台的 skill 加载机制**。

```
部署方式：
┌──────────┐    下载 skill 文件    ┌──────────────┐
│ GitHub   │ ───────────────────►  │ AI代理平台    │
│ 发布包    │                      │ (Hermes等)    │
└──────────┘                      └──────┬───────┘
                                         │
                                   ┌─────▼─────┐
                                   │ 身份注入    │
                                   │ + 记忆管理   │
                                   └─────┬─────┘
                                         │
                              ┌──────────┴──────────┐
                              │    AI 引擎（任意）    │
                              └─────────────────────┘
```
