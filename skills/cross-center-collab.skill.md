# 跨中心协作规范

## 协作原则

1. **并行不等待**：独立任务同时派出，不串行
2. **角色清晰**：每个中心有明确职责边界
3. **结果汇聚**：各自产出后由指挥官统一整合

## 标准协作流程

```
任务到达
  ↓
指挥官拆解为独立子任务
  ↓ (并行派出)
┌────┬────┬────┐
│中心A│中心B│中心C│ ← 各自独立执行
└────┴────┴────┘
  ↓ (全部完成后)
指挥官汇总 → 质量检查 → 交付
```

## 适用场景

- 多维度调研（市场+技术+合规同步）
- 内容生产（文案+设计+审核并行）
- 系统建设（架构+开发+测试）
